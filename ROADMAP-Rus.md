# Daily Time Tracker - Roadmap

> Живой документ с уже выпущенными возможностями, активными планами и идеями, от которых сознательно отказались.

---

## Уже выпущено

### Серия (огонек)
> Выпущено в [v1.3.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.3.0)

- Огонек и счетчик серии в шапке popup.
- Цветовые уровни по длине серии.
- Сброс после пропущенного полного дня.
- Минимальный порог для серии: 5 минут.

### Экспорт в CSV / JSON
> Выпущено в [v1.3.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.3.0)

- CSV для таблиц.
- JSON для бэкапа и переноса данных.

### Очистка / сброс данных
> Выпущено в [v1.3.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.3.0)

- Сбросить сегодня.
- Очистить историю.
- Полный сброс.

### Импорт из JSON
> Выпущено в [v1.6.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.6.0)

- Режимы Merge и Replace.
- Валидация и предпросмотр перед применением.

### Недельная сводка
> Выпущено в [v1.7.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.7.0)

- Отдельная кнопка `Неделя` в шапке popup.
- Общее время за последние 7 дней.
- Среднее в день.
- Самый активный день.
- Мини-график на 7 дней.

### Режимы Compact / Full
> Выпущено в [v1.7.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.7.0)

- Compact mode делает popup короче для повседневного использования.
- Full mode показывает все сессии, полную историю и weekly toggle.
- Режим popup можно менять из Settings.

### Настраиваемый порог паузы
> Выпущено в [v1.7.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.7.0)

- Можно выбрать, сколько секунд паузы допускается до завершения текущей сессии.
- Доступные пресеты: `15 / 30 / 60 / 120 сек`.
- Настройка находится в Settings и сразу влияет на поведение трекинга.

### Streak Shields + Keep Streak
> Выпущено в [v1.7.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.7.0)

- Доступно до `4 щитов` на календарный месяц.
- Пропущенный день может потратить один щит вместо сброса серии.
- Щиты сохраняют текущую серию, но не увеличивают ее.
- Режим `Keep streak` сохраняет текущую серию без роста, пока он включен.

### Daily Goal
> Выпущено в [v2.0.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v2.0.0)

- Пользовательская цель на день в Settings.
- Прогресс до цели показывается в popup.
- Значение `0` отключает цель, не удаляя саму настройку.

### Keep History Forever
> Выпущено в [v2.0.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v2.0.0)

- Добавлена опция `Forever` рядом с обычными лимитами хранения истории по месяцам.
- Историю можно хранить без автоматической очистки.
- Настройка остается опциональной, поэтому поведение по умолчанию остается легким.

### Оптимизация LocalStorage
> Выпущено в [v2.0.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v2.0.0)

- Хранилище разделено на отдельные ключи для текущего дня, сессий и компактной истории.
- Сессии старых дней больше не хранятся в полном виде бесконечно.
- Миграция со старого формата происходит автоматически.

### Разделение runtime по каналам + поток обновлений
> Выпущено в [v2.0.1](https://github.com/vvertax/DailyTimeTracker/releases/tag/v2.0.1)

- `release`, `test` и `dev` теперь используют более понятные channel-specific runtime файлы.
- Канал можно переключать из Settings с подтверждением и последующей перезагрузкой Spotify.
- В hosted runtime встроены ручная проверка обновлений и действия, привязанные к тексту версии.

### Top Tracks Today
> Выпущено в `v2.1.0`

- Необязательный блок `Top tracks` под основным временем за день в popup.
- Показывает самые часто проигрываемые треки только за текущий день.
- В Settings можно выбрать отображение `1`, `2` или `3` треков.
- По умолчанию блок выключен.
- Прослушивание засчитывается только после минимум `20` секунд реального воспроизведения, чтобы быстрая промотка в конец не давала фейковый count.

### Yesterday vs Today
> Выпущено в `v2.1.0`

- Альтернативный подрежим внутри экрана `Неделя`.
- Быстрое сравнение с предыдущим днем.
- Показывает значения `Today`, `Yesterday` и `Difference` в одном месте.

### Performance Mode
> Выпущено в `v2.1.0`

- В Settings появились режимы `Default` и `Lightweight`.
- Уменьшает нагрузку popup и визуальных эффектов на слабых CPU/GPU.
- Помогает сделать popup более отзывчивым во время длинных сессий.

### Улучшение отзывчивости popup
> Выпущено в `v2.1.0`

- Popup теперь избегает большего числа лишних перерисовок.
- Недельная сводка и сводные секции эффективнее переиспользуют уже отрендеренный контент.
- Включение `Keep streak` больше не создает эффект визуального падения серии.

## В планах

> Сейчас отдельный релизный milestone еще не зафиксирован, но следующие идеи считаются реально полезными и остаются в рабочем scope.

- `Longest Session Today`
  - Короткая сводка по самой длинной сессии за текущий день.
- `Goal Forecast`
  - Оценка, будет ли достигнута дневная цель при текущем темпе и сколько времени еще осталось.
- `Top Artists Today`
  - Агрегация по артистам поверх уже существующих дневных данных по трекам.
- `History Filters`
  - Простые фильтры вроде дней с выполненной целью, дней с длинными сессиями или дней выше заданного порога.
- Небольшие UI-полировки и технические обновления продолжаются между крупными фичами.

## Отменено

Следующие идеи были пересмотрены и сознательно исключены из планов, потому что для этого проекта их практическая ценность признана недостаточной.

### Сравнение с прошлой неделей
> Отменено - признано недостаточно полезным для дальнейшей разработки.

### Разбивка по дням недели
> Отменено - признано недостаточно полезным для дальнейшей разработки.

### Индикатор темпа прослушивания
> Отменено - признано недостаточно полезным для дальнейшей разработки.

### Тепловая карта по часам
> Отменено - признано недостаточно полезным для дальнейшей разработки.

### Граница дня с учетом часового пояса
> Отменено - признано недостаточно полезным для дальнейшей разработки.

## Вне текущего scope

| Идея | Почему отложено |
|---|---|
| Облачная синхронизация | Потребует инфраструктуру авторизации; для одного устройства LocalStorage достаточно. |
| Аналитика по трекам | Доступные playback API недостаточно надежны для архивной аналитики по трекам. |
| Синхронизация между окнами | LocalStorage здесь не является надежным каналом синхронизации между экземплярами. |
| Тяжелые библиотеки графиков | Слишком большой вес для легкого расширения. |
