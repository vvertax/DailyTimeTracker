# Daily Time Tracker - Roadmap

> Живой документ с уже выпущенными возможностями, активными планами и идеями, от которых сознательно отказались.

---

## Уже выпущено

### Серия (огонек)
> Выпущено в [v1.3.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.3.0)

- Огонек и счетчик серии в шапке popup.
- Цветовые уровни по длине серии.
- Сброс после пропущенного полного дня.
- Минимальный порог для серии: 5 минут.

### Экспорт в CSV / JSON
> Выпущено в [v1.3.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.3.0)

- CSV для таблиц.
- JSON для бэкапа и переноса данных.

### Очистка / сброс данных
> Выпущено в [v1.3.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.3.0)

- Сбросить сегодня.
- Очистить историю.
- Полный сброс.

### Импорт из JSON
> Выпущено в [v1.6.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.6.0)

- Режимы Merge и Replace.
- Валидация и предпросмотр перед применением.

### Недельная сводка
> Выпущено в [v1.7.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.7.0)

- Отдельная кнопка `Неделя` в шапке popup.
- Общее время за последние 7 дней.
- Среднее в день.
- Самый активный день.
- Мини-график на 7 дней.

### Compact / Full Popup Modes
> Выпущено в [v1.7.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.7.0)

- Compact mode делает popup короче для повседневного использования.
- Full mode показывает все сессии, полную историю и weekly toggle.
- Режим popup переключается из Settings.

### Configurable Pause Threshold
> Выпущено в [v1.7.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.7.0)

- Пользователь может выбрать, сколько секунд паузы допускается до завершения текущей сессии.
- Доступные пресеты: `15 / 30 / 60 / 120 сек`.
- Настройка находится в Settings и сразу влияет на поведение трекинга.

### Streak Shields + Keep Streak
> Выпущено в [v1.7.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v1.7.0)

- Доступно до `4 щитов` на календарный месяц.
- Пропущенный день может потратить один щит вместо сброса серии.
- Щиты сохраняют текущую серию, но не увеличивают ее.
- Режим `Keep streak` сохраняет текущую серию без роста, пока он включен.

### Daily Goal
> Выпущено в [v2.0.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v2.0.0)

- Пользовательская цель на день в Settings.
- Прогресс до цели показывается в popup.
- Значение `0` отключает цель, не удаляя саму настройку.

### Keep History Forever
> Выпущено в [v2.0.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v2.0.0)

- Добавлена опция `Forever` рядом с обычными лимитами хранения истории по месяцам.
- Историю можно хранить без автоматической очистки.
- Настройка остается опциональной, поэтому поведение по умолчанию остается легким.

### LocalStorage Optimization
> Выпущено в [v2.0.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v2.0.0)

- Хранилище было разделено на отдельные ключи для текущего дня, сессий и компактной истории.
- Сессии старых дней больше не хранятся в полном виде бесконечно.
- Миграция со старого формата происходит автоматически.

### Runtime Refactor
> Выпущено в [v2.0.0](https://github.com/vvertax/DailyTimeTracker/releases/tag/v2.0.0)

- Hosted runtime был разделен на `main.mjs`, `core.mjs`, `dtt_optimization.mjs` и `version.json`.
- Это упростило поддержку и деплой обновлений.

### Channel Runtime Isolation
> Выпущено в [v2.0.1](https://github.com/vvertax/DailyTimeTracker/releases/tag/v2.0.1)

- `release`, `test` и `dev` теперь используют более понятную channel-based runtime структуру.
- Test и Dev проще обновлять без правок основного release entry flow.
- Общая инфраструктура вроде API и storage optimization при этом может оставаться общей.

## Вне текущего скоупа

| Идея | Почему отложено |
|---|---|
| Облачная синхронизация | Потребует инфраструктуру авторизации; для одного устройства LocalStorage достаточно. |
| Аналитика по трекам | Доступные playback API недостаточно надежны для архивной аналитики по трекам. |
| Синхронизация между окнами | LocalStorage здесь не является надежным каналом синхронизации между экземплярами. |
| Тяжелые библиотеки графиков | Слишком большой вес для легкого расширения. |
