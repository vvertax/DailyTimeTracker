# Daily Time Tracker для Spicetify

Скрипт считает, сколько времени вы слушаете Spotify каждый день, и показывает таймер в верхней панели.

> [!NOTE]
> Этот код писался с помощью ChatGPT.

![Превью](./preview.png)

## Возможности

- **Таймер в верхней панели:** Счётчик в формате `ЧЧ:ММ:СС` или `ДД:ЧЧ:ММ:СС`.
- **Более точный трекинг сессий:** Сессии опираются на реально прослушанное время, а не на сырое время паузы, и аккуратнее завершаются, когда воспроизведение действительно останавливается.
- **Настраиваемый порог паузы:** В Settings можно выбрать `15 / 30 / 60 / 120 сек`.
- **Подробная история:** Popup с сессиями за сегодня и архивом по дням.
- **Топ-треки за день:** Необязательный блок под общим временем за день с самыми часто проигрываемыми треками текущего дня. Прослушивание засчитывается только после минимум `20` секунд реального воспроизведения.
- **Недельная сводка + Yesterday vs Today:** Кнопка `Неделя` показывает последние 7 дней, среднее, лучший день, мини-график и быстрый режим сравнения со вчерашним днём.
- **Режимы Compact / Full:** Compact делает popup короче; Full показывает все сессии, всю историю и недельный переключатель.
- **Режим производительности:** Можно выбрать `Default` или `Lightweight`, чтобы уменьшить нагрузку popup и визуальных эффектов на слабых системах.
- **Дневная цель:** В Settings можно задать цель по времени прослушивания в минутах и видеть прогресс прямо в popup.
- **Хранение истории + Forever:** Историю можно хранить от `1` до `6` месяцев или отключить автоочистку через `Forever`.
- **Огонёк серии:** Счётчик серии появляется от `2+` подряд идущих qualifying days.
- **Щиты серии + Keep streak:** До `4` пропущенных дней в календарном месяце могут автоматически защитить серию, а опция `Keep streak` сохраняет текущую серию без роста.
- **Длинная прогрессия серии:** Дополнительные цветовые уровни для длинных streak.
- **Каналы Release / Test / Dev:** Канал можно переключать из Settings. Release стабильнее, Test может содержать баги, а Dev доступен только отдельным тестерам.
- **Ручная проверка обновлений:** Обновления можно проверить из Settings, а текст версии работает как быстрый триггер проверки.
- **Экспорт в CSV / JSON:** Выгрузка всей сохранённой истории и текущего дня из Settings.
- **Импорт из JSON:** Восстановление или объединение предыдущего JSON-экспорта с предпросмотром и валидацией.
- **Очистка и сброс:** Можно сбросить сегодня, очистить историю или сделать полный сброс с подтверждением.
- **Переключение языка:** `RU` и `EN` прямо внутри popup.
- **Пользовательские бейджи:** Необязательный серверный бейдж рядом с заголовком popup.
- **Удалённые runtime-обновления:** Marketplace-loader подтягивает основной runtime с `https://vvertax.site/dtt/ext/main.mjs` и проверяет обновления через `https://vvertax.site/dtt/ext/version.json`.

## Текущий интерфейс

- **Виджет в верхней панели:** Показывает текущее время за сегодня.
- **Шапка popup:** Заголовок, необязательный бейдж, огонёк серии, mode-aware кнопка `Неделя` и Settings.
- **Раздел Today:** Общее время за день, необязательный прогресс цели, необязательный блок top tracks, сессии за сегодня и сохранённая история.
- **Раздел Week:** Недельная сводка со средним, лучшим днём, мини-графиком и подрежимом `Yesterday vs Today`.
- **Панель Settings:** Язык, канал, режим popup, режим производительности, top tracks, количество top tracks, порог паузы, щиты серии, `Keep streak`, дневная цель, срок хранения истории, длинная прогрессия серии, видимость бейджа, экспорт, импорт, destructive reset actions, ручная проверка обновлений и текст версии.

## Установка

### Через Spicetify Marketplace

1. Откройте Spicetify Marketplace.
2. Найдите `Daily Time Tracker` во вкладке **Extensions**.
3. Нажмите **Install**.

### Ручная установка

1. Скачайте `DailyTimeTracker.js` [здесь](marketplace/DailyTimeTracker.js).
2. Поместите файл в папку расширений Spicetify:
   `%AppData%\spicetify\Extensions` на Windows или `~/.config/spicetify/Extensions` на Linux/macOS.
3. Выполните:

```powershell
spicetify config extensions DailyTimeTracker.js
spicetify apply
```

## Какие данные хранятся

Скрипт использует `Spicetify.LocalStorage`:

- `dtt_today_v3`
- `dtt_sessions_v1`
- `dtt_history_v2`
- `dtt_language_v1`
- `dtt_daily_goal_seconds_v1`
- `dtt_history_retention_months_v1`
- `dtt_history_retention_forever_v1`
- `dtt_popup_mode_v1`
- `dtt_performance_mode_v1`
- `dtt_pause_threshold_seconds_v1`
- `dtt_streak_v1`
- `dtt_streak_control_v1`
- `dtt_long_streak_progression_enabled_v1`
- `dtt_badge_visibility_v1`
- `dtt_session_track_counts_v1`
- `dtt_top_tracks_visible_v1`
- `dtt_top_tracks_display_count_v1`
- `dtt_channel_v1`
- `dtt_test_notice_seen_v1`
- `dtt_version_v1`

## Примечания

- Очистка истории затрагивает видимый сохранённый архив, но серия всё равно считается по сырым историческим данным.
- Данные top tracks привязаны к текущему дню и сбрасываются при начале нового дня.
- После переключения канала требуется перезагрузка Spotify.
- Архитектура с hosted runtime позволяет выпускать фиксы быстрее, без ожидания Marketplace review.
- Скрипт зависит от DOM-структуры Spotify и Spicetify, поэтому изменения интерфейса Spotify могут потребовать фиксов.
